# Component Specs

## Buttons

- Primary: solid neutral/accent, medium emphasis
- Secondary: subtle border, no heavy fill
- Tertiary: text-only, for low emphasis actions
- Destructive: danger tone only when irreversible

## Inputs

- Height: 40 (compact) or 44 (comfortable)
- Border-first style, not glow-first
- Focus: visible 2px ring with high contrast
- Placeholder: secondary text only; never instructional overload

## Cards/Panels

- Border + slight surface tint > heavy shadow
- Keep one purpose per card
- Header optional; avoid redundant labels

## Tables/Lists

- Right-align numeric columns
- Sticky header for long lists
- Row hover subtle; selected state explicit

## Chat/Agent UI

- Thread max width for readability
- Tool/action metadata collapsible
- Streaming text should not shift layout aggressively
- Separate user/system/agent visually by spacing + label, not bright colors
