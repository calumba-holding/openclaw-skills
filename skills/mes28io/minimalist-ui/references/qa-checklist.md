# QA Checklist (Pass/Fail)

## Visual

- [ ] One clear primary action per screen
- [ ] Consistent spacing rhythm (8pt scale)
- [ ] Max one accent color used intentionally
- [ ] No decorative-only UI elements

## Usability

- [ ] First-time user can identify next action in <5s
- [ ] Errors explain recovery path
- [ ] Empty states suggest meaningful action
- [ ] Keyboard focus is always visible

## Accessibility

- [ ] Text contrast meets WCAG AA minimum
- [ ] Interactive targets are comfortably clickable
- [ ] Form fields have labels (not placeholder-only)

## Engineering Readiness

- [ ] States defined: default/hover/focus/disabled/loading/error
- [ ] Responsive behavior defined
- [ ] Token usage explicit (type/space/color)
- [ ] No ambiguous “make it pretty” instructions
